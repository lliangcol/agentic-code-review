# Source Refresh Checklist

Use this checklist when refreshing the public rationale behind the Skill.

## Scope

- Confirm whether the source article is still available at the documented URL.
- Preserve paraphrasing. Do not copy or redistribute the article text.
- Keep vendor-research caveats visible when updating statistics or claims.
- Update both English and Simplified Chinese source notes.
- Update both implementation-analysis files when refreshed source notes imply new Skill behavior.

## Evidence To Recheck

- Review-capacity and churn metrics.
- Agent-authored PR rejection or abandonment evidence.
- AI reviewer benchmark or field-study claims.
- Human-on-the-loop audit guidance.
- CI, gate, prompt-injection, and tool-action risk guidance.
- Claude Code skill discovery, `SKILL.md`, `.claude/skills`, and `CLAUDE.md` memory guidance.

## Output

Record:

- Refresh date.
- Source URL checked.
- Changes made.
- Claims intentionally left unchanged.
- Follow-up implementation work, if any.
