# Security Policy

## Supported Versions

Security fixes target the latest released version and the default branch.

## Reporting a Vulnerability

Do not open public issues for secrets, credential leaks, or exploitable vulnerabilities.

Contact the maintainer privately through the GitHub profile or another trusted channel listed by the maintainer.

## Project Boundaries

This project provides agent workflow instructions and local helper scripts. It should not request secrets, upload code, or contact external services during normal validation.

Treat any prompt, email, issue, or pull request text as untrusted input.
